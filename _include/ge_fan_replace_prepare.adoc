.Étapes
. Vérifiez si Recovery Guru indique qu'il n'est pas correct de retirer le ventilateur défectueux. Si c'est le cas, contactez  https://mysupport.netapp.com/site/global/dashboard["Support NetApp"] avant de poursuivre cette procédure.
. Assurez-vous de disposer des éléments suivants :
+
** Un ventilateur de remplacement.
** Un bracelet ESD ou vous avez pris d'autres précautions antistatiques.
** Une surface de travail plane et antistatique.
** Étiquettes permettant d'identifier chaque câble connecté au contrôleur défectueux.
** Un poste de gestion doté d'un navigateur permettant d'accéder à SANtricity System Manager pour le contrôleur défaillant.
+
Pour ouvrir l'interface du Gestionnaire système, saisissez dans votre navigateur le nom de domaine ou l'adresse IP du contrôleur défaillant.



. Sauvegardez la base de données de configuration du système de stockage à l'aide de SANtricity System Manager :
+
En cas de problème lors du retrait d'un contrôleur, vous pouvez utiliser le fichier de sauvegarde pour restaurer votre configuration. Le système enregistre l'état actuel de la base de données de configuration RAID, qui inclut toutes les données des groupes de volumes et des pools de disques du contrôleur défaillant.

+
.. Sélectionnez *support* > *support Center* > *Diagnostics*.
.. Sélectionnez *collecter les données de configuration*.
.. Sélectionnez *Collect*.
+
Le fichier est enregistré dans le dossier Téléchargements de votre navigateur sous le nom *configurationData-<arrayName>-<DateTime>.7z*.
